# Taste

This directory captures personal preferences and style conventions — the things that are important to how output looks and feels, but don't fit neatly into a workflow.

Think of it as **the difference between "how to do it" and "how it should look when done."** Workflows describe process; taste describes the end result.

## Contents

- **writing-style.md** — prose conventions: tone, structure, sentence-level preferences
- **sql-style.md** — SQL formatting, naming, and structuring conventions
- **file-naming.md** — naming conventions for files and directories
- **corpus/** — sample documents showing the style in practice (creative, docs, emails, posts)

## How to Use

Agents should consult taste files when generating output to match the right aesthetic. Taste files are reference material — read when style matters, not for procedural steps.
