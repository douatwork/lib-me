# Core

This directory holds the foundational layer: personal identity, operating values, and heuristics.

- **profile.md** — personal context the user chooses to share. If left blank or missing, the rest of the system works fine.
- **beliefs.md** — core beliefs and worldview that shape decision-making.
- **heuristics.md** — mental models and decision shortcuts for common situations.

These files are the "who" and "why." The other directories (stack, taste, workflows) are the "what" and "how" — they function independently if core is sparse or missing.

## Reading Order

1. `readme.md` — you are here
2. `profile.md` — who the user is (optional)
3. `beliefs.md` — what the user believes
4. `heuristics.md` — how the user thinks

Load these only when the context calls for it. Don't assume they're needed for every task.
