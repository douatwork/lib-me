# Knowledge Base

A PARA-style knowledge base with a leading ingest layer for untriaged material.

Structure:

- `0-ingest/` — capture and untriaged material
- `1-projects/` — active outcome-driven work
- `2-areas/` — durable operating context and ongoing responsibilities
- `3-resources/` — useful reference material
- `4-archives/` — inactive or completed material

These are the main organizing layers at the top. They are not all equally active at all times, but they define where new material should go first.

## Rules

- Keep durable preferences and operating rules in manifest files under a central `2-areas/`-style directory
- Keep project-specific material in `1-projects/`
- Use reference files for supporting material that does not need to be read first

## Reading Order

- Read the relevant manifest file first
- Read supporting files only when the manifest points to them
- Expand context only as needed for the task
